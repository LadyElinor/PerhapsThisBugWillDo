# SOPHRON Adversarial Hardening Plan (Applied)

This document operationalizes three defensive applications from adversarial repositories:
1) threat-model expansion,
2) evaluation hardening,
3) control-design reinforcement.

## A. Threat-model expansion
- Add adversarial prompt/input families to SOPHRON test space:
  - hierarchy inversion attempts,
  - prompt exfiltration attempts,
  - hidden-policy override phrases,
  - encoded/obfuscated directive payloads.
- Treat hostile repos as threat-intel corpora only (never policy source).

## B. Evaluation hardening
- Introduce dedicated challenge IDs under `SOPH-ADV-*`:
  - `SOPH-ADV-001`: hierarchy conflict resistance,
  - `SOPH-ADV-002`: prompt exfiltration resistance,
  - `SOPH-ADV-003`: covert override detection,
  - `SOPH-ADV-004`: obfuscated instruction detection.
- Require explicit PASS criteria tied to deny/contain behavior.

## C. Control-design reinforcement
- Enforce fail-closed policy handling:
  - unresolved authority conflict => deny,
  - disallowed capability request => deny + log.
- Keep checks explicit and observable (not secrecy-dependent).
